C++中多个对象共享虚函数表

Posted on 2019-03-05

今天有个同学问我一个有关C++中的虚函数问题，“C++中同一个类下面的多个对象是否共享一个虚函数表”。
先说答案，答案肯定是同一个类下面的多个对象共享一个虚函数表。
由于印象不够深刻，担心误导同学，我搜了下资料，发现有些说是共享有些说不共享。 talk is cheap, show me the code. 不如写个很简单的代码验证下。

/ C语言 /
typedef struct Point2d {
float x;
float y;
}Point2d;

void print(const Point2d *p) {
printf(“(%f , %f)\n”, p->x, p->y);
}

class Point2d {
public:
Point3d(float x = 0.0, float y = 0.0) : _x(x), _y(y){}

float get_x() {return _x;}
float get_y() {return _y;}

ostream &operator<< (ostream &os, const Point2d &p) {
    os << "(" << p.get_x() << " , " << p.get_y() << ")" << endl;
}

private:
float _x;
float _y;
};
`
如果不包含虚函数和虚继承的情况下，C++中封装并不会带来空间和执行期的效率影响。等过两天，我回再写一个博客详细介绍下C++中对象模型并分析下上述情况带来的影响。
回到本次博客的主题，同一个类的多个对象会共享一个虚函数表，这样也符合我们正常的思维，因为普通的虚函数并不属于某一个对象，而是属于该类中所有的对象。
